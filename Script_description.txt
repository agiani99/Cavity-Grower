## Design philosophy

| Stage | What happens |
|---|---|
| **Seed** | Single C atom placed *exactly* on the DU coordinate (cavity centre) |
| **Grow** | Iterative beam-search: for each beam member, sample attachment points ├Ś fragment actions |
| **Embed** | Free ETKDGv3 + MMFF, then translate so atom 0 stays on anchor; each retry also applies a random rotation, giving diverse cavity orientations |
| **Filter** | Hard: ÔëĄ 30% clash with protein atoms, ÔëĄ 25% outside cavity sphere; then QED Ôëą threshold, SA ÔëĄ threshold |
| **Score** | `composite = 0.40┬ĚQED + 0.35┬ĚSA_norm + 0.25┬Ě(1Ôłĺclash) Ôłĺ 0.30┬Ěout_frac` |
| **Harvest** | When HAC reaches `--target-min ÔÇŽ --target-max` (default 16ÔÇô20) |

Updated the pharmacophore term so it also penalizes ÔÇťunmatchedÔÇŁ ligand features (extra donors/acceptors/charges/aromatics that donÔÇÖt find a complementary pocket site).

**What changed**
- In de_novo_cavity_growth.py, `ph4` is no longer just `n_match / n_total`.
- New formula (clamped to [0, 1]):

  `raw = (n_match - w_unmatched * n_unmatched) / n_total`  
  `ph4 = clamp(raw, 0, 1)`

So if the ligand keeps adding features that donÔÇÖt complement the cavity, `ph4` drops instead of staying ÔÇťokÔÇŁ.

**New knob**
- `--ph4-unmatched-weight` (default `0.5`)
  - `0.0` reproduces the old behavior (reward-only, no penalty)
  - Higher values penalize unmatched features more strongly
